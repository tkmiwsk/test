# キャラクター演出コンセプト設計

> 作成日: 2026-03-06
> 目的: 画面デザインに入る前に「キャラクターを画面内でどう魅せるか」の方針を定義する

---

## この文書の位置づけ

既存ドキュメントでは以下が定義済み：
- **キャラの人格**（characters.md） → 何者か
- **キャラのセリフ**（dialogue-patterns.md） → 何を言うか
- **キャラの見た目**（character-design-sheet.html） → どんな外見か

**この文書で定義するもの → 画面の中で「どう魅せるか」**

キャラクターはUIパーツではない。ユーザーが「会いに来る理由」そのもの。
この設計が甘いと、ただのアイコン付きチャットボットに見えてしまう。

---

## 1. キャラクター表示の3つのモード

画面内でキャラクターが登場する場面を3段階に整理する。

| モード | 名称 | キャラの表示サイズ | 役割 | 使用画面 |
|--------|------|------------------|------|---------|
| **L** | ステージモード | 画面の50%以上 | キャラクターが「主役」として登場する瞬間 | スプラッシュ、診断結果、キャラ切替演出、初回API連携 |
| **M** | 対話モード | 画面の20〜30% | キャラクターが「アドバイザー」として語りかける場面 | ホームのひとこと、投稿フィードバック |
| **S** | 常駐モード | アイコンサイズ（32〜48px） | キャラが「いつもそばにいる」ことを感じさせる | タブバー横、通知バッジ、データカード内 |

### なぜ3段階に分けるか

- **常にLモードだと**：キャラが邪魔になり、データが見づらい
- **常にSモードだと**：キャラへの愛着が生まれない。ただのアイコン
- **メリハリがあることで**：Lモードの登場が「イベント感」を持ち、Sモードの存在が「安心感」になる

---

## 2. ステージモード（L）の演出設計

**コンセプト: 「キャラクターの見せ場」をつくる**

キャラが画面の中心に立つ場面。この瞬間が「スクショを撮りたくなる瞬間」であり、バズ導線の起点になる。

### 2-1. 登場パターン

| パターン | 演出 | 使用場面 |
|---------|------|---------|
| **ドラマチック登場** | 暗転→光の粒子→キャラがフェードイン→セリフ表示 | 診断結果、キャラ切替演出 |
| **静かな存在感** | キャラが画面にすでにいて、ユーザーを「待っていた」ように見える | API連携画面、スプラッシュ |
| **サプライズ** | 通常のUI操作中に突然Lモードで出現（例: 過去最高記録時） | ホーム画面での特別イベント |

### 2-2. キャラの「立ち位置」

```
┌─────────────────────────┐
│                         │
│     ┌───────────┐       │  ← キャラは画面上部〜中央
│     │           │       │     ユーザーの目線の先に配置
│     │  キャラ    │       │
│     │  イラスト   │       │
│     │           │       │
│     └───────────┘       │
│                         │
│  ┌─────────────────────┐│  ← セリフは常にキャラの下
│  │    セリフ吹き出し     ││     キャラを見て→セリフを読む
│  └─────────────────────┘│     の視線誘導を作る
│                         │
│     [CTAボタン]          │  ← アクションは最下部
│                         │
└─────────────────────────┘
```

**原則: 上からキャラ → セリフ → アクションの順で視線が流れる**

### 2-3. スクショ映え設計

ステージモードの画面は「SNSでシェアされる前提」で設計する。

| 要素 | 方針 |
|------|------|
| 背景 | キャラのテーマカラーを使ったグラデーション。余白をたっぷり取る |
| キャラの表示 | 上半身〜全身。表情がはっきり見えるサイズ |
| セリフ | 短く、インパクトのある一言。長文にしない |
| ブランド要素 | 画面下部にアプリ名やURLをさりげなく入れる（シェア時の帰属） |

## 3. 対話モード（M）の演出設計

**コンセプト: 「キャラが自分に話しかけてくれている」感覚をつくる**

ホーム画面や投稿フィードバックでキャラがコメントする場面。日常的に最も多く触れるモード。

### 3-1. 表示フォーマット

```
┌─────────────────────────┐
│                         │
│  ┌──────┐              │
│  │キャラ │ キャラ名       │
│  │アイコン│ ─────────    │
│  └──────┘              │
│  ┌─────────────────────┐│
│  │                     ││
│  │  セリフテキスト       ││  ← 吹き出し形式
│  │                     ││
│  └─────────────────────┘│
│                         │
└─────────────────────────┘
```

### 3-2. アイコンの表情連動

**対話モード最大のポイント：キャラアイコンがセリフの内容に応じて表情を変える**

| セリフの内容 | ヨミの表情 | バッサーの表情 | ヌクルの表情 |
|------------|-----------|-------------|------------|
| 良い数字の報告 | 口角がわずかに上がる | ニヤッと笑う | にっこり |
| 注意・改善提案 | 真剣な目 | 腕組み・眉上げ | 少し心配そう |
| 褒め・レア感情 | 照れ（目をそらす） | 照れ隠し（横を向く） | 嬉しくて目がキラキラ |
| エラー・障害 | 申し訳なさそう | 舌打ち（自分に苛立つ） | 不安そう |

**この「表情の変化」が、キャラをただのアイコンから"感情のある存在"に変える鍵。**

### 3-3. セリフの表示演出

| 要素 | 方針 |
|------|------|
| テキスト表示 | 一文字ずつ表示（タイピングアニメーション）。キャラが「今しゃべっている」感覚 |
| 「……」の演出 | ヨミの沈黙は「間」として0.5秒のウェイトを入れる |
| セリフの長さ | 対話モードでは最大3行まで。長い分析はタップで展開 |
| タイムスタンプ | セリフの下に「3時間前」のような時間表示。リアルタイム感 |

## 4. 常駐モード（S）の演出設計

**コンセプト: 「いつもそばにいる」を感じさせる**

小さいが存在感がある。キャラがアプリ全体を「見守っている」雰囲気をつくる。

### 4-1. 常駐の表現方法

| 場所 | 表現 |
|------|------|
| ナビゲーションバー | キャラアイコンがアプリロゴの代わりに左上に常駐 |
| データカード | サマリーカードの端にキャラのミニ表情を添える（数字が良い→笑顔、悪い→心配顔） |
| 通知 | プッシュ通知のアイコンがキャラアイコン。通知文もキャラの口調 |
| ローディング | データ取得中にキャラのミニアニメ（ヨミ: タブレットを見てる / バッサー: 腕組みで待ってる / ヌクル: うとうと） |

### 4-2. テーマカラー連動

選択中のキャラによって、アプリ全体のアクセントカラーが変わる。

| キャラ | アクセントカラー | 変わる対象 |
|--------|----------------|----------|
| ヨミ | シアンブルー #0EA5E9 | ナビバーのアクティブ色、ボタン、リンク、グラフのハイライト |
| バッサー | クリムゾン #F43F5E | 同上 |
| ヌクル | エメラルド #10B981 | 同上 |

**効果: キャラを選んだ実感が「色」として常に目に入る。「自分だけのアプリ」感が生まれる。**

## 5. キャラごとの「画面の空気」の違い

同じ画面構成でも、キャラによって画面の雰囲気が変わるべき。

| 要素 | ヨミ | バッサー | ヌクル |
|------|------|---------|-------|
| 背景の雰囲気 | 暗めのネイビー。夜空・デジタル感 | ダークグレー。都会的・ストリート感 | 明るいクリーム。朝の光・自然感 |
| データの見せ方 | 数値を大きく。グラフ重視 | 結論を太字で。改善点を赤字で強調 | 数値は控えめ。「頑張ったね」の文脈で包む |
| 吹き出しのデザイン | シャープな角。薄い枠線 | 角丸だがしっかりした枠。赤のアクセント | もこもこした雲のような形。枠線なし |
| アニメーション | 静的。データがスッと現れる | 力強い。スライドインで現れる | ふわっと。フェードインでやさしく現れる |

## 6. 「スクショしたくなる瞬間」の設計

バズ導線の根幹。キャラクターの魅力が最大化する「決定的瞬間」を意図的に設計する。

### 6-1. スクショポイント一覧

| # | 瞬間 | キャラ | 何が起きるか | なぜシェアされるか |
|---|------|--------|------------|----------------|
| 1 | 診断結果の表示 | 全キャラ | Lモードで登場＋決めセリフ | 「私はヨミタイプだった！」→ 診断系コンテンツ |
| 2 | バッサーの辛口コメント | バッサー | Mモードだが、特に辛辣なセリフ時に表情が際立つ | 「バッサーにボコボコにされた」→ 共感・笑い |
| 3 | ヨミのレア感情表現 | ヨミ | 普段無表情のヨミが笑った瞬間にLモードに切り替わる | レア感 → 「ヨミが笑った！」と報告したくなる |
| 4 | ヌクルの本気モード | ヌクル | 普段穏やかなヌクルが敬語になる瞬間にLモードに切り替わる | ギャップ → 「ヌクルが怒った……」と報告したくなる |
| 5 | 過去最高記録更新 | 全キャラ | 特別なエフェクト＋キャラのLモード登場 | 成果の共有欲求 |
| 6 | キャラ切替演出 | 2体（退場＋登場） | 退場キャラの別れのセリフ→登場キャラの歓迎 | ドラマとして楽しい |

### 6-2. 「Lモードへの昇格」ルール

通常の対話モード（M）からステージモード（L）に切り替わる条件を定義する。
これにより、**普段小さいキャラが突然大きく出てくる「サプライズ感」**が生まれる。

| トリガー | 演出 |
|---------|------|
| 過去最高記録を更新した | 画面全体がキャラカラーに光る → キャラがLモードで登場 |
| ヨミが感情を見せた（レアセリフ発動） | 通常のMモード表示→フワッとLモードに切り替わる |
| バッサーが初めて褒めた | 画面が一瞬止まる→バッサーがLモードで「……悪くないよ」 |
| ヌクルが敬語になった（本気の心配） | 画面のトーンが少し暗くなる→ヌクルがLモードで真剣な顔 |
| 連続投稿の節目（7日、30日、100日） | お祝い演出＋キャラLモード |

## 7. キャラとデータの関係性

**原則: データはキャラの「根拠」であって、キャラの「代わり」ではない**

| NG例 | OK例 |
|------|------|
| 数値テーブルを表示して、端にキャラアイコンを置く | キャラがセリフで数値に言及し、詳細はタップで展開 |
| グラフの横にキャラが無言で立っている | キャラがグラフの特定ポイントを指差して「ここ、見て」と言う |
| データ画面にキャラを飾りとして配置 | キャラがデータの「語り手」として画面をナビゲートする |

### データの見せ方の優先順位

```
1. キャラのセリフ（結論・感情を伝える）
   ↓
2. キーとなる数字1つ（エンゲージメント率 5.1%）
   ↓
3. 詳細データ（タップで展開）
```

**まずキャラの言葉で「良いニュースか悪いニュースか」を伝え、次に数字で裏付ける。データダンプは最後。**

## 8. イラスト素材として必要なバリエーション

画面設計に入る前に、イラストレーターに依頼すべきバリエーションを整理する。

### キャラ1体あたりに必要な素材

| カテゴリ | 内容 | 点数 | 用途 |
|---------|------|------|------|
| **全身イラスト** | 立ちポーズ（通常） | 1点 | Lモード: 診断結果、キャラ選択 |
| **全身イラスト** | 特別ポーズ（喜び/怒り/照れ） | 3点 | Lモード: 特別イベント時 |
| **表情アイコン** | 8表情バリエーション（48px〜120px対応） | 8点 | Mモード/Sモード: 対話・常駐 |
| **ミニアニメ** | ローディング用ポーズ | 1点 | Sモード: データ読み込み中 |
| **シェア用** | 背景付きの正方形カード用イラスト | 1点 | SNSシェア画像 |
| | **1体あたり合計** | **14点** | |
| | **3体合計** | **42点** | |

### 追加で検討すべき素材

| 素材 | 内容 | 判断ポイント |
|------|------|------------|
| キャラ掛け合いイラスト | 2体が並んだ構図（切替演出用） | v1で必要か？ テキストのみでも成立するか？ |
| 背景パターン | キャラごとの世界観背景 | Lモードの背景はグラデーションのみ？ イラスト背景？ |
| エフェクト素材 | 光の粒子、紙吹雪など | CSSアニメで代替可能か？ |
